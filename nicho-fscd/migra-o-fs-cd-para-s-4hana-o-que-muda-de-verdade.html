Migração FS-CD para S/4HANA: o que muda de verdade

Na conversão para S/4HANA, o FS-CD continua como FI-CA porém sobre o novo data model. Pontos críticos:

  • Universal Journal (ACDOCA): a parte G/L do FS-CD passa a espelhar no universal journal; open items ficam em DFKKOP.
  • Business Partner: obrigatório — o modelo de cliente legado (KNA1) deve ser convertido.
  • Customizing de eventos (FQEVENTS): regras de derivação precisam de revalidação.

O erro mais caro é tratar migração FS-CD como 'copy-paste' de tabelas. É reestruturação de modelo de dados.

⚠️ Revisão técnica pendente: Refinar com consultor SAP: não dizer que 'todos os dados FS-CD espelham no ACDOCA' — open items (DFKKOP) continuam em tabela própria; só o G/L vai para ACDOCA.
